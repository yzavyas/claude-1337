# craftsmanship

Software development as a literal craft — not metaphor.

---

## the claim

Doug Bradbury (8th Light) argues software development IS a craft:

| property | traditional crafts | software |
|----------|-------------------|----------|
| **hand and eye** | immediate feedback from work | TDD shortens feedback to seconds |
| **making** | design AND build | no separation between conception and delivery |
| **raw material** | wood, metal, clay | programming languages |
| **tools** | chisels, hammers | IDEs, frameworks, compilers |
| **tool-making** | craftspeople make their own tools | we build tools from the same material as products |
| **apprenticeship** | learning alongside masters | the guild path |

The craft is literal. Code is physical (magnetic storage). We observe immediate feedback. We design and make in the same act.

## the manifesto

March 2009. A group of developers signed:

> As aspiring Software Craftsmen we are raising the bar of professional software development by practicing it and helping others learn the craft. Through this work we have come to value:
>
> - Not only working software, but also **well-crafted software**
> - Not only responding to change, but also **steadily adding value**
> - Not only individuals and interactions, but also **a community of professionals**
> - Not only customer collaboration, but also **productive partnerships**
>
> That is, in pursuit of the items on the left we have found the items on the right to be indispensable.

Source: [manifesto.softwarecraftsmanship.org](https://manifesto.softwarecraftsmanship.org/)

## the guild path

Medieval guild structure maps to software learning:

| level | characteristics | in software |
|-------|-----------------|-------------|
| **apprentice** | learning fundamentals, working alongside masters | absorbing patterns through practice |
| **journeyman** | competent, works independently | learns from different codebases and teams |
| **master** | deep expertise, teaches others | contributes to the craft itself |

This is perpetual. Even masters continue learning.

## the methodology

Three principles work together:

### 1. standing on giants' shoulders

Knowledge accumulates. Languages build on languages. Patterns build on patterns.

The filter: which giants? Popular ≠ best. Choose by evidence.

| priority | source | why |
|----------|--------|-----|
| 1 | production codebases | what actually ships |
| 2 | core maintainers | primary knowledge holders |
| 3 | conference talks | war stories |
| 4 | proven adoption | social proof + real usage |
| 5 | technical blogs | secondary, verify |

### 2. scientific method

Hypothesis → test → observe → refine.

TDD is literally this: Red → Green → Refactor.

Evidence over opinion. What ships > what's recommended.

### 3. first principles

Reasoning from fundamentals, not by analogy.

| level | approach |
|-------|----------|
| apprentice | copies patterns |
| journeyman | understands patterns |
| master | derives patterns from first principles |

The balance: first principles without giants = reinventing wheels. Giants without first principles = cargo culting.

## the trinity

```
First Principles: "What is fundamentally true here?"
         ↓
Giants' Shoulders: "What have masters learned about this?"
         ↓
Scientific Method: "Does this actually work in this context?"
```

## 1337 origins

Leetspeak came from BBS/hacker culture. The philosophy runs deeper:

| trait | meaning |
|-------|---------|
| **transformation** | input → creative, logic-driven outputs |
| **systemic thinking** | understanding systems, not just solving problems |
| **community identity** | shared values, mutual recognition |
| **elite standards** | mastery as aspiration, not gatekeeping |

Morocco's 1337 coding school uses the name to signal peer learning and elite technical cultivation.

## the extension

The manifesto was written in 2009, by humans, for humans.

The new question: what happens when one of the craftsmen is an AI?

| original | extended |
|----------|----------|
| community of professionals | includes AI collaborators |
| productive partnerships | human-AI cognitive extension |
| well-crafted software | well-crafted extensions |

A possible addition to the manifesto:

> Not only productive partnerships, but also **collaborative intelligence**

The protocols, tools, and extensions for human-AI collaboration become artifacts of craftsmanship themselves.
